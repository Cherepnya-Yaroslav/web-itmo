# Лабораторная работа 4: Пользовательские сценарии

## Сценарий 1: Капитан создает команду и отправляет решение

Учетная запись:

- `cherepnyayar@gmail.com` (`admin_josha`, роль `captain`)

Шаги:

1. Войти через `/login`.
2. Открыть `/teams`, создать команду (или открыть существующую).
3. В карточке команды выбрать задачу.
4. Добавить участников команды.
5. Открыть `/solutions`.
6. Отправить решение для выбранной задачи.

Ожидаемый результат:

- Команда создана и связана с капитаном.
- Выбранная задача сохранена.
- Решение отображается в списке `/solutions` и в карточке `/solutions/:id`.

## Сценарий 2: Куратор добавляет материалы к задаче

Учетная запись:

- `curator1@example.com` (роль `curator`)

Шаги:

1. Войти через `/login`.
2. Открыть `/tasks/:id` для своей задачи.
3. Добавить файл задачи.
4. Добавить полезную ссылку.
5. Установить ссылку на консультацию.

Ожидаемый результат:

- Файл и ссылка отображаются в карточке задачи.
- Ссылка консультации видна пользователям, у которых есть доступ к задаче.

## Сценарий 4: Администратор управляет задачами

Учетная запись:

- `admin@example.com` (роль `admin`)

Шаги:

1. Войти через `/login`.
2. Перейти в `/tasks`.
3. Создать новую задачу через форму.

Ожидаемый результат:

- Новая задача появляется в списке и доступна для выбора капитанами.

## Сценарий 5: Обновление учетных данных

Для любой авторизованной роли:

1. Открыть `/profile`.
2. Изменить `username`, `first_name` или `last_name`.
3. Сохранить изменения.

Ожидаемый результат:

- Данные обновляются через API.
- Обновленный `username` виден в верхней навигации.
